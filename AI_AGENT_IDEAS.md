# AI Agent Ideas — Next Iteration

Нижче список практичних ідей для подальшого розвитку ШІ-агента після поточної реалізації.

## 1) Smart Disambiguation
- Якщо агент знаходить кілька лідів-кандидатів (наприклад, однакові імена), він не виконує дію одразу.
- Замість цього задає уточнення: «Якого саме ліда ви маєте на увазі: #12 чи #47?»

## 2) Explain Mode (Low Confidence)
- Якщо confidence < порогу, агент коротко пояснює, що саме зрозумів.
- Приклад: «Я зрозумів це як команду додати нотатку до ліда #15. Підтвердити?»

## 3) Action Suggestions
- Після кожної відповіді агент повертає 2–3 релевантні наступні кроки.
- Приклади:
  - після `list`: `покажи гарячі`, `статистика`, `пошук`;
  - після `analyze`: `додай нотатку`, `познач QUALIFIED`, `передати в sales`.

## 4) Context TTL + Session Summary
- Контекст очищується по TTL (вже додано), але можна також зберігати коротке summary сесії.
- Це допоможе відновлювати діалог після перезапуску (через Redis/DB).

## 5) Confirmation Policy Matrix
- Гнучка матриця підтвердження за типом дії:
  - always confirm: delete, transfer;
  - conditional: edit stage (тільки для terminal stage);
  - no confirm: show/list/search.

## 6) Hybrid NLU (Rules + LLM Intent Repair)
- Спочатку працюють rule-based патерни.
- Якщо confidence низький — LLM робить “intent repair” у strict JSON schema.

## 7) Agent Analytics
- Логувати: intent, confidence, execution status, latency, fallback usage.
- Метрики:
  - % успішно розпізнаних команд,
  - % команд із уточненням,
  - частка fallback до AI.

## 8) Safety Rails
- Блокувати небезпечні/неповні команди без обов’язкових entities.
- Наприклад: `видали ліда` без `lead_id` → тільки уточнення, без виконання.

## 9) Multilingual Persona
- Автоматичне визначення мови (UA/EN), але системна відповідь українською за замовчуванням.
- Додати перемикач мови в settings.

## 10) Voice UX Quality
- Показувати “confidence badge” для розпізнавання голосу.
- Для низької якості аудіо — пропонувати швидке переформулювання кнопками.
